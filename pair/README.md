## 앱 이름: 우아한 백코
## 카테고리: 페어 프롬프트 릴레이

### 배포 링크
[우아한 백코 바로가기](https://gemini.google.com/share/5402c6455c4d)

### 이 앱을 만든 이유
- 어떤 문제/불편함을 해결하려고 했나요?

> 매일 알고리즘 문제 풀이에 앞서 문제를 선택하는 시간 소요 해결과 작성된 코드의 완성도를 직관적으로 피드백 받을 수 있도록 아래 서비스를 구현하였습니다.

- 이 앱을 언제, 어떻게 사용할 건가요?   
> 알고리즘 문제 선택시에 사용하여 해당 문제를 해결하고, 작성된 코드에 대한 더 정확하고 시각적인 피드백이 요구될 때 사용됩니다.

### 주요 기능
#### 최소 기능 요구 사항 정의 선행

---

1. 백준 문제집 링크를 사용자가 직접 등록한다.
2. 백준 문제집을 기반으로 매일 1문제 랜덤 추천 문제를 제공한다.
3. 추천 문제는 사용자가 설정한 옵션(조건)에 포함하는 문제만 추천된다.
    - 알고리즘 유형 선택 혹은 무작위
    - 난이도 범위 선택 슬라이드 형식바 제공(브론즈5 ~ 다이아1)
4. 문제 정보를 조회
    1. 문제 추천시에 아래 정보를 모두 한 페이지에 제공한다.
        - 문제의 상세 조건과 본문, 그리고 예시 입출력을 함께 제공한다.
        - 알고리즘이 뭔지 보일지 안 보일지(정보제공유무, 즉 숨김 처리 기능 제공)
        - 난이도가 보일지 안 보일지(정보제공유무, 즉 숨김 처리 기능 제공)
        - 구현한 코드에 대해 AI피드백을 받을 수 있는 텍스트박스 제공(해당 기능에 대해서는 아래 상세 작성 예정)
5. 문제집 내에 연관문제 번호 제공
    - 사용자가 등록한 문제집 내의 모든 문제를 크롤링하여, 연관된 문제 번호를 제공한다.
    - 이때, 반드시 크롤링을 하여 추천하도록 한다. (기타 AI 기반 지식 활용을 하지 않고, 크롤링된 데이터만을 기반으로 추천한다.)
6. AI 기반 피드백 기능
    - 적절한 자료구조를 사용하였는지, 불필요한 변수 사용, 더 효율적인 방법 제시
    - 작성된 코드의 알고리즘 시간 복잡도 제공
    - 작성된 코드가 작동하는 흐름을 인포그래픽(자료구조, 그래프, 탐색, 등등)로 제공하여, 사용자가 작성한 코드의 문제점을 발견할 수 있도록 도와줌

### AI 기능
- 어떤 AI 기능을 추가했나요?
> 작성된 코드에 대한 AI 기반 피드백을 추가하였습니다. 피드백 요청시 동작하는 프롬포트는 다음과 같습니다.
```
사용자가 백준 알고리즘 문제 "${state.currentProblemContext.title}"에 대해 코드를 작성했습니다.
문제 설명 일부: ${state.currentProblemContext.description}

사용자 코드:
\`\`\`
${code}
\`\`\`

아래 5가지 항목에 대해 명확하고 상세하게 한국어로 피드백을 작성해주세요.
1. 적절한 자료구조를 사용하였는지에 대한 평가
2. 불필요하게 사용된 변수나 로직이 있는지 확인
3. 더 효율적이거나 깔끔한 알고리즘/방법 제시
4. 작성된 코드의 알고리즘 시간 복잡도(Big-O) 분석
5. (중요) 해당 알고리즘의 동작 방식을 시각적으로 쉽게 이해할 수 있는 인포그래픽(텍스트 다이어그램)을 제공해주세요.
- 외부 라이브러리(Mermaid 등)는 절대 사용하지 마세요. (문법 에러 발생 위험)
- 반드시 유니코드 선 문자(├, └, │, ─, →, ↓ 등)와 ASCII Art를 활용하여 트리, 그래프, 배열 상태 변화 등을 텍스트로 직관적으로 그려주세요.
- 다이어그램은 반드시 \`\`\`diagram ... \`\`\` 코드 블록으로 감싸주세요.
```

- AI 기능이 앱에서 어떤 역할을 하나요?
> 사용자가 작성한 코드에 대해 더 나은 개선점을 분석하여 제공합니다. 문제 자체에 대한 권장 풀이보다는 사용자의 코드의 개선점을 위주로 분석하여 제공합니다. 추가적으로 그래프, 트리, DP와 같은 알고리즘은 동작 방식을 시각적으로 쉽게 이해할 수 있도록 인포그래픽으로 제공합니다.

> 처음 시도에서 mermaid.js 라이브러리를 통한 시각화 도구를 제공하였지만, 안정적으로 출력되지 않은 케이스들이 빈번하게 발견되어, 안정성을 위해 라이브러리 사용이 아닌 유니코드 문자를 이용하여 시각 자료를 제공하였습니다.
